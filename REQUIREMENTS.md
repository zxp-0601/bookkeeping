# 订单利润统计系统 — 需求文档 (Requirements Document)

## 1. 项目概述

一个轻量级的 Web 应用，用于记录和管理电商订单的收支明细，自动计算合作费（分成）和单笔利润。支持人工录入、智能识别粘贴文本、表格文件批量导入、增删改查、批量操作、分页查询等功能。

系统采用**单页应用** (SPA) 架构，后端使用 Node.js 原生 HTTP Server，数据以 JSON 文件持久化存储，无需数据库。

---

## 2. 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 后端 | Node.js (纯 `http` 模块) | 原生 HTTP Server，无框架依赖 |
| 前端 | 原生 HTML + CSS + JavaScript | 单页应用，无框架依赖 |
| 数据存储 | `records.json` | JSON 文件存储所有订单记录 |
| 表格解析 | SheetJS (`xlsx.full.min.js`) | CDN 加载，用于解析 `.xlsx` 文件 |

## 3. 功能需求

### 3.2 数据统计面板

| 💰 销售额 | 总销售额 | ¥xxx.xx |
| 📉 成本 | 总成本 | ¥xxx.xx |

| 📄 订单数 | 订单总数 | |

### 3.3 智能识别 (Smart Parse)

**输入**：用户粘贴一段非结构化的中文文本（例如从聊天记录中复制的订单信息）。

**要求识别的字段**：

| 字段 | 来源句式示例 | 说明 |
|---|---|---|
| 数量 (quantity) | `卖了N把/个/件/双/只/条/箱/份/套/单` | 未匹配时默认为 1 |
| 销售额 (sales_amount) | `卖了N块钱/元/米` / `卖了××，卖了N块钱` | 优先从 `卖了N块钱/元/米` 提取金额 |
| 客户名称 (customer_name) | `收货人:XXX` / `收件人:XXX` / `联系人:XXX` / `姓名:XXX` | |
| 渠道 (channel) | `闲鱼` / `淘宝` / `拼多多` / `京东` 等关键词 | 从 `（闲鱼）卖了` 等句式提取 |
| 平台昵称 (platform_nickname) | `昵称是XXX` / `昵称:XXX` | |
| 规格/颜色 (product_spec) | `颜色是XXX` / `规格:XXX` / `颜色黑色小花` | |
| 日期 (date) | `2024年1月5日` | 未匹配时使用当天日期 |
| 收件信息 (recipient_address) | 拼接 `收货人:XXX` + `手机:XXX` + `地址:XXX` | 用 `|` 分隔 |

**自动填入的默认值**：

| 字段 | 默认值 | 说明 |
|---|---|---|
| 成本 (cost) | `数量 × 6` | 每次修改数量立即更新，输入框只读 |
| 快递费 (shipping_fee) | `3.00` | 识别后直接填入表单 |
| 手续费 (service_fee) | `销售额 × 0.006` | 识别时自动填入，输入销售额时自动填入 |

### 3.4 新增订单表单

**必填字段**（所有必填项未填时提交按钮禁用）：

- 数量 (默认空)
- 销售额 (¥) — 必须 > 0
- 成本 (¥) — readonly，自动 = 数量 × 6
- 快递费 (¥) — 默认 3.00
- 手续费 (¥) — 默认 销售额 × 0.006
- 完成情况 (下拉选择：未完成/已发货/已完成)
- 客户名称
- 平台昵称
- 渠道
- 日期

**选填字段**：

- 快递单号
- 规格
- 收件信息

**只读自动计算字段**（所有 4 个数值字段都填写后才计算）：

- 合作费 = (销售额 - 成本 - 快递费 - 手续费) / 2
- 单笔利润 = 合作费 (二者相等)

**提交行为**：

- 提交前自动检查并填充默认值（成本、快递费、手续费）
- 验证全部通过后将数据 POST 到 `/api/records`
- 提交成功后清空表单，刷新列表和统计

**提示信息**：表单底部以红色粗体显示 "⚠️ 请人工核对订单数据，避免出错！"

### 3.6 筛选功能

支持 3 种筛选条件组合使用：
1. **日期范围**：开始日期 + 结束日期
2. **状态**：下拉选择（全部/未完成/已发货/已完成）
3. **关键词搜索**：模糊匹配客户名称或平台昵称（不区分大小写）

### 3.7 批量操作

- 表格每行复选框 + 表头全选
- 选中后出现批量操作栏
- 支持：批量修改状态（未完成/已发货/已完成）、批量删除

### 3.8 修改订单

- 点击"修改"按钮弹出修改弹窗
- 可修改全部字段
- 保存后自动重新计算合作费和利润
- 使用 `PUT /api/records?id=xx` 接口

### 3.9 删除订单

- 点击"删除"按钮，确认后执行删除
- 使用 `DELETE /api/records?id=xx` 接口

### 3.10 各渠道统计

两个并排面板分别展示各渠道的销售额和利润统计。

### 3.11 表格导入

**功能**：用户选择 CSV 或 Excel (.xlsx) 文件，自动解析并映射列后导入数据库。

**流程**：
1. 点击"📂 导入表格"按钮打开导入弹窗
2. 选择文件（支持 `.csv` 和 `.xlsx`）

## 4. API 接口

### 4.1 `GET /api/records` — 查询订单列表

**查询参数**：`from`, `to`, `status`, `search`

**返回**：按 ID 倒序排列的订单数组，附带计算字段 `profit`

### 4.2 `POST /api/records` — 新增订单

**请求体**：全部订单字段（含 `recipient_address`）

**返回**：`{ ok: true, data: { id, profit, cooperation_fee } }`

### 4.3 `PUT /api/records?id=xx` — 修改订单

**请求体**：可只传需要修改的字段

### 4.4 `DELETE /api/records?id=xx` — 删除订单

### 4.5 `POST /api/records/batch` — 批量导入

**请求体**：`{ records: [...] }`

### 4.6 `GET /api/summary` — 统计数据

**返回**：按渠道分组统计 + 总计

## 5. 数据模型

```json
{
  "id": 1,
  "quantity": 2,
  "sales_amount": 22.00,
  "cost": 12.00,
  "shipping_fee": 3.00,
  "service_fee": 0.13,
  "cooperation_fee": 3.44,
  "status": "已完成",
  "customer_name": "张三",
  "platform_nickname": "糊涂小朋友",
  "tracking_number": "SF1234567890",
  "product_spec": "粉色小菊花",
  "recipient_address": "张三 | 13800138000 | 安徽省宿州市砀山县",
  "channel": "闲鱼",
  "date": "2024-01-15",
  "created_at": "2024/1/15 14:30:00"
}
```

**计算公式**：
- `cooperation_fee = (sales_amount - cost - shipping_fee - service_fee) / 2`
- `profit = cooperation_fee`
- `手续费 = 销售额 × 0.006`
- `成本 = 数量 × 6`
