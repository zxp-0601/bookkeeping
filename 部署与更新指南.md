## 📌 准备工作

### 什么是 Git、GitHub 和部署平台？

| 名词 | 说明 |
|------|------|
| **Git** | 你电脑上的"时光机"，记录每次代码修改 |
| **GitHub** | 网上的代码仓库，类似"云盘"，存放你的代码 |
| **Railway** | 免费的"托管平台"，把你的代码变成真正的网站 |

**流程：** 你修改代码 → 用 Git 推送到 GitHub → Railway 自动检测更新 → 网站自动更新

## 🔧 手动操作方式（备用）

### 1️⃣ 打开终端（Command + 空格 → 搜索"终端"）

### 2️⃣ 进入项目文件夹

```bash
cd /Users/zcl/Desktop/bookkeeping
```

### 3️⃣ 一条命令提交并推送

```bash
git add .
git commit -m "修改内容说明"
git push
```

> 把 `"修改内容说明"` 改成你这次改了什么，比如 `"修复导入表格bug"`。

### 4️⃣ 等待 Railway 自动部署

推送成功后，Railway 会自动检测到新代码并重新部署。大概 1-3 分钟后，公网地址就会更新。

## 🔄 如何查看是否部署成功？

1. 打开 Railway：https://railway.app/dashboard
2. 点击项目 `bookkeeping`
3. 查看 Deployment 状态，显示绿色 ✅ 说明部署成功
4. 点击域名即可访问

## ⚠️ 常见问题

### Q：git push 时报错 "Authentication failed"
**原因：** Token 不对

**解决：** 用正确的 GitHub Token（已保存在远程地址中，一般不会出错）

### Q：推送到 GitHub 后网站没变化
**原因：** Railway 部署需要时间，通常 1-3 分钟

**解决：** 去 Railway 后台查看 Deploy 日志

### Q：怎么知道自己 GitHub 的用户名？
打开 https://github.com，右上角头像旁边显示的就是用户名
